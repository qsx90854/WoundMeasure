# 立體視覺三角測距系統：完整程式設計流程

## 系統概述

本系統以同一支相機在位置略有差異的情況下拍攝兩張影像為輸入，讓使用者在第一張影像（影像 A）上點選任意一個查詢點，系統自動在第二張影像（影像 B）上找到精度優於 0.5 pixel 的對應點，再利用已知大小與形狀的 pattern 所估計的相機相對位姿（R, T）以及已知的相機內參（K），執行三角測距，輸出該點的三維世界座標, 與相機光心的歐式距離, 還有與ArUco平面的歐氏距離。


## 第一階段：前置準備與輸入

程式啟動後，首先載入相機內參矩陣 K（焦距 fx、fy 與主點 cx、cy）以及畸變係數（若相機已預先校正則可為零）在calibration_result.json內。開啟相機後會顯示左右圖, 可選定當前影像為右圖, 左圖為camera live影像, 內參都使用同一組
。場景中必須包含至少一個已知實體尺寸與形狀的 pattern，且為 ArUco 標記板，其三維點座標在 pattern 自身座標系下已知, 預設參數ArUco尺寸為33x33mm。


## 第二階段：利用 Pattern 估計相對位姿（R, T）

對影像 A 與影像 B 分別執行 pattern 特徵點偵測，取得各自的影像二維座標與對應的 pattern 三維座標配對。將每張影像的配對資料連同相機內參 K 送入 `cv2.solvePnP()`，分別求得各影像相對於 pattern 座標系的旋轉向量 rvec 與平移向量 tvec，再透過 `cv2.Rodrigues()` 將旋轉向量轉換為 3×3 旋轉矩陣 R1、R2 與平移向量 t1、t2。相機 A 到相機 B 的相對旋轉為 `R_rel = R2 @ R1.T`，相對平移為 `T_rel = t2 - R_rel @ t1`，baseline 則取 `‖T_rel‖` 的歐式距離。此相對位姿即為後續立體幾何的基礎。場景中拍攝到的複數ArUco都是同一平面, 請用這個前提來提升估計精準度



## 第三階段：立體校正（Stereo Rectification）

取得 R_rel 與 T_rel 後，呼叫 `cv2.stereoRectify()` 計算兩張影像的校正矩陣 R1_rect、R2_rect 以及校正後的投影矩陣 P1、P2 和視差-深度映射矩陣 Q。接著對影像 A 與影像 B 分別呼叫 `cv2.initUndistortRectifyMap()` 產生重映射查找表，再用 `cv2.remap()` 將兩張影像轉換為校正後版本（imgA_rect、imgB_rect）。

立體校正的核心目的是將兩張影像的對極線（epipolar line）對齊為水平方向，使得影像 A 上任意一點 (x, y) 在影像 B 上的對應點必定位於同一水平掃描線 y 上。這個性質將後續的二維搜尋問題降維為一維問題，大幅提升匹配速度與精度。


## 第四階段：使用者點選查詢點

使用者透過 UI 在原始影像 A 上點選一個像素座標 p1_orig = (u, v)。程式利用第三階段產生的重映射查找表，將此座標轉換到校正後空間中的對應座標 p1_rect = (x1, y1)。這個轉換可以透過對查找表進行逆映射，或直接以 `cv2.undistortPoints()` 搭配校正矩陣完成。後續所有的匹配與三角化計算均在校正後的影像空間中進行。


## 第五階段：多尺度 Patch 匹配（核心匹配模組）

### 5-1 擷取參考 Patch

以 p1_rect = (x1, y1) 為中心，在 imgA_rect 上擷取一個大小為 patch_size × patch_size 的局部影像區塊（建議 patch_size = 21，確保涵蓋足夠的局部紋理資訊）。若查詢點過於靠近影像邊緣導致 patch 越界，則向內縮減搜尋範圍或提示使用者重新選點。

### 5-2 確定搜尋範圍

根據場景預期的最大深度範圍，估算最大視差 max_disp（即 x1 − x2 的最大可能值）。在極線校正後的影像 B 上，對應點必定位於同一水平線 y1，x 方向的搜尋範圍為 [x1 - max_disp, x1 + search_margin]，其中 search_margin 為對小量位移的容錯量（通常 10–20 px）。

### 5-3 粗搜尋（ZNCC 滑動匹配）

在搜尋範圍內以整像素步進，對每個候選位移 d 擷取 imgB_rect 上以 (x1 + d, y1) 為中心的 patch，計算與參考 patch 的零均值歸一化互相關係數（ZNCC）：

> ZNCC(d) = Σ[(IA - Ā)(IB - B̄)] / √[Σ(IA - Ā)² · Σ(IB - B̄)²]

ZNCC 的值域為 [-1, 1]，1 代表完全相同，對光照均勻變化（亮度偏移、對比度縮放）具有良好的魯棒性，適合相同相機在不同曝光條件下拍攝的情形。記錄所有候選位移對應的 ZNCC 分數，取最高分的整像素位置 d_peak 為初始估計。

### 5-4 亞像素精化（拋物線擬合）

取 d_peak 鄰近三個點 S(d_peak - 1)、S(d_peak)、S(d_peak + 1) 的 ZNCC 分數，以二次拋物線擬合求極值位置：

> d* = d_peak − [S(d+1) − S(d−1)] / {2 × [S(d+1) − 2S(d) + S(d−1)]}

此亞像素修正量通常可使最終誤差降至 0.1–0.3 pixel，穩定達到 < 0.5 pixel 的精度目標。最終對應點座標為 p2_rect = (x1 + d*, y1)。

### 5-5 多尺度策略（選用，提升速度與魯棒性）

若影像解析度較高或預期視差範圍較大，可先在降採樣影像（如 1/4 尺寸）上執行粗搜尋，取得視差初值後再回到原始解析度在窄範圍內精搜尋，並執行亞像素精化。此策略可顯著縮短計算時間，同時減少陷入局部極值的機率。


## 第六階段：匹配信心度驗證

在輸出對應點之前，必須通過三項信心度篩選，防止紋理薄弱或重複紋理區域產生錯誤匹配。

**ZNCC 峰值閾值**：要求最高分 S(d_peak) > 0.85。分數過低代表局部紋理過弱或場景在兩張影像間發生遮擋或形變。

**唯一性比率（Uniqueness Ratio）**：要求最高分與第二高分（需與 d_peak 距離超過 2 px）之比率 > 1.2。比率接近 1.0 代表分數分佈平坦，存在多個可能的對應點，匹配不可靠。

**局部梯度強度**：計算參考 patch 的影像梯度強度（Sobel 或 Laplacian），若強度低於設定閾值（例如純色牆面、無紋理桌面），則標記為「紋理不足，無法可靠匹配」，並向使用者提示換點。

若上述任一條件不滿足，系統可嘗試擴大 patch 尺寸至 31×31 重新匹配，或改用對弱紋理更魯棒的 Census Transform 替代 ZNCC，再次進行驗證。


## 第七階段：三角測距

通過信心度驗證後，取得校正空間中的對應點對 p1_rect = (x1, y1) 與 p2_rect = (x2, y1)，視差為 disp = x1 − x2。利用視差-深度關係式估算深度：

> Z = f_rect × baseline / disp

其中 f_rect 為校正後的焦距（可從 P1 矩陣的 [0,0] 元素取得），baseline 為第二階段計算的 ‖T_rel‖。

對於更高精度的需求，應採用直接線性變換（DLT）三角化。將 p1_rect 與 p2_rect 分別以齊次座標表示，連同校正後的投影矩陣 P1、P2 送入 `cv2.triangulatePoints()`，取得齊次四維座標後除以第四個分量，即得三維世界座標 P_world = (X, Y, Z)。DLT 方法在兩台「相機」光軸不完全平行時依然有效，並可自然處理亞像素輸入。

最後計算重投影誤差（Reprojection Error）：將 P_world 重新投影回兩張影像，計算投影點與原始點的像素距離，作為輸出結果的品質指標。重投影誤差應低於 1 pixel，理想情況下低於 0.5 pixel。


## 輸出結果

系統最終輸出包含：使用者選取點對應的三維世界座標 (X, Y, Z)、影像 B 上的對應像素座標 p2_orig（映射回原始影像空間）、ZNCC 匹配信心分數、重投影誤差。在 UI 層面，可在影像 A 上標記查詢點，在影像 B 上標記對應點，並在兩點之間繪製連線，同時以文字顯示估計的三維距離與信心指標。
