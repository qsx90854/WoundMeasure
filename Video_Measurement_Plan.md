# 針對影片多視角測量 3D 深度與距離的建議流程

您的想法非常切合攝影測量 (Photogrammetry) 的核心概念！既然您已經錄製了環繞待測物拍攝的影片，並且上面有固定的 ArUco 標籤作為基準，我們可以將這個問題轉換為**「雙視角/多視角立體三角測量 (Stereo Triangulation)」**。

以下是為您量身打造的建議流程，這套流程可以直接沿用並微調我們目前開發的系統來達成：

## 1. 核心概念：建立「以待測物為中心的絕對座標系」
目前我們的 live 程式是「以左相機為原點 (0,0,0)」來計算深度。但在影片中相機是一直在移動的，因此最好的做法是：
* **將 ArUco 標籤所在的平面定義為「世界座標系 (World Coordinate)」的原點 ($Z=0$ 平面)**。
* 這樣一來，任何量測出來的 3D 座標 $(X, Y, Z)$ 中，**$Z$ 值就代表該點距離待測物表面的「絕對高度/深度」**。
* 兩點的「深度差距」就純粹是 $|Z_1 - Z_2|$，而「空間距離」則是 3D 歐式距離。

---

## 2. 建議的操作流程 (Workflow)

### 步驟一：從影片擷取兩個「關鍵幀 (Keyframes)」
1. 程式改為讀取影片檔 (Video File)。
2. 使用者在畫面上播放影片，並手動挑選兩個**視角差異夠大**的影格（例如相差 15~30 度）。
3. 分別按下「鎖定影像 A」與「鎖定影像 B」。*(視角差異越大，三角測量的深度精度越高！)*

### 步驟二：計算相機在兩幀的絕對姿態
1. 系統分別在「影像 A」與「影像 B」中偵測 ArUco 標籤。
2. 透過 `cv2.solvePnP`，計算出**「影像 A 相機」相對於 ArUco 平面**的姿態 ($R_A, T_A$)。
3. 同理，計算出**「影像 B 相機」相對於 ArUco 平面**的姿態 ($R_B, T_B$)。
4. 有了絕對姿態，我們就能推導出兩台相機之間的相對姿態 (Baseline 與 Epipolar 幾何) 供後續匹配使用。

### 步驟三：手動選取目標點
1. 使用者在「影像 A」上，使用左鍵點擊**目標點 1**，中鍵點擊**目標點 2**。

### 步驟四：特徵匹配 (Feature Matching)
1. 系統利用影像 A 點擊的特徵，沿著影像 B 的極線 (Epipolar Line) 進行 ZNCC/SIFT 區塊匹配，找出這兩點在「影像 B」中精確的亞像素 (Sub-pixel) 對應點。
2. 由於影片拍攝可能會遇到視角形變，這裡會啟動我們之前寫好的 ECC (Affine) 變形補償來提升匹配精度。

### 步驟五：3D 三角測量與數據計算
1. 擁有「影像 A 座標」、「影像 B 座標」以及兩台相機相對於 ArUco 平面的「投影矩陣 ($P_A, P_B$)」。
2. 使用 `cv2.triangulatePoints` 將 2D 點轉化為 3D 座標 $(X_1, Y_1, Z_1)$ 與 $(X_2, Y_2, Z_2)$。
3. **輸出結果：**
   * **點 1 絕對深度**：$Z_1$
   * **點 2 絕對深度**：$Z_2$
   * **相對深度差距**：$|Z_1 - Z_2|$
   * **3D 歐式距離**：$\sqrt{(X_1-X_2)^2 + (Y_1-Y_2)^2 + (Z_1-Z_2)^2}$

---

## 4. 進階優化：充分利用影片的「中間幀 (Intermediate Frames)」
只使用兩幀的確浪費了影片帶來的豐富連續資訊。若要將 RT (Rotation/Translation) 姿態與深度量測的準確度推向極致，我們可以引入 **Structure from Motion (SfM)** 與 **連續追蹤** 的概念：

### 優化一：相機姿態 (RT) 軌跡平滑化
* **單幀解算的缺點**：ArUco 單幀解算容易因為影像雜訊產生姿態跳動 (Jitter)。
* **多幀優化方案**：
  由於影片的相機運動是連續且平滑的，我們可以利用**滑動平均 (Moving Average)** 或 **卡爾曼濾波器 (Kalman Filter)** 來平滑相機的 3D 軌跡。甚至可以利用相鄰幀的特徵點匹配 (Essential Matrix) 來約束 ArUco 估計出的 RT，確保相機軌跡在 3D 空間中是完美平滑的。

### 優化二：光流法連續追蹤 (Optical Flow Tracking)
* **單次寬基線匹配的缺點**：若影像 A 與影像 B 視角差異過大 (Wide Baseline)，ZNCC 或 SIFT 的區塊匹配容易因為嚴重的透視形變而失敗。
* **多幀優化方案**：
  當您在影像 A 點擊目標後，系統不直接跳到影像 B 去找，而是利用 **Lucas-Kanade (LK) 光流法** (`cv2.calcOpticalFlowPyrLK`)。
  因為相鄰影格 (Frame $i$ 到 $i+1$) 的變化極小，光流法可以「黏」著特徵點，一路從影像 A 穩穩地追蹤經過數十張中間幀，直到抵達影像 B。這能完美解決大角度形變導致配對失敗的問題！

### 優化三：多視角立體三角測量 (N-View Triangulation)
* **雙視角的缺點**：只用 A, B 兩幀計算，如果剛好某幀的 2D 座標差了 1 像素，3D 深度就會有明顯誤差。
* **多幀優化方案**：
  既然我們透過光流法追蹤了目標點在 $N$ 張中間幀的 2D 座標，且我們擁有這 $N$ 幀的相機絕對姿態 $(P_1, P_2, ..., P_N)$。
  我們可以將「雙視角三角測量」升級為 **「多視角三角測量 (Multi-View Triangulation)」**！透過建立超定方程組 (Overdetermined System) 並使用 SVD 奇異值分解求最小平方法解，這樣可以將 $N$ 張畫面的雜訊全部平均掉，深度的準確度會呈幾何級數上升！

## 5. 開發實作方案建議

我們可以直接將目前的程式複製一份，命名為 `depth_measure_video_mode.py`，並做以下調整：
1. **輸入源更改**：將 `cv2.VideoCapture` 改為讀取影片檔，並加入影片播放、暫停、前後格跳轉的 UI 介面。
2. **座標系轉換**：修改 PnP 解算邏輯，將基準座標系從「相機 A」改為「ArUco 所在的 Global Plane」。
3. **引入 LK 光流追蹤**：當使用者點選目標點後，系統自動往後播放並即時追蹤該點軌跡。
4. **升級為 N-View 三角測量**：收集軌跡上的多個相機姿態與 2D 座標，計算最精確的 3D $(X,Y,Z)$ 座標。
5. **UI 顯示擴充**：在畫面上顯示目標點的追蹤軌跡，並輸出 `Delta Z` (深度差距)。
